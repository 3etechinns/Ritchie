Ritchie Stock Exchange™

Welcome!

"Ritchie" is an Alternative Trading System (ATS) that is created, built, run, and managed by the open-source community.1

In the spirit of "The Open Source Way,"2 we believe that a market--and its data--must be free, open, and transparent to be "fair and orderly:"

  • Designed for Algorithmic Trading.
  • No "non-display" fees. Ritchie is designed from the ground up for non-display.
  • Affordable colocation fees. Colocation is affordable at Ritchie. Your algos can be run on a device as small as a Raspberry PiTM.
  • Random matching algorithm. Not a FIFO order queue, an array where orders are matched randomly.
  • Real-time tick data are free. Easily accessed through our API.
  • Open Order Book/DOM.
  • Open API. Designed for algorithmic trading and quantitative analysis.
  • Open Architecture. The inner workings of the exchange are open for public scrutiny.
  • Open Order Arrays.
  • Free FIX adapter
  • Open Source!

All that being said, there are limits:

  • Anonymity is strictly enforced. The identity of market participants is not public information.
  • Commissions are charged. An industry-standard fee schedule is equitably applied to all transactions.

In sum, the guiding principle at Ritchie is that of a meritocracy:

In a meritocracy, the best ideas win. In a meritocracy, everyone has access to the same information. Successful work determines which projects rise and gather effort from the community.2

What is the Ritchie technology stack?

  • Terminal! Some form of shell. Ritchie is designed for algorithmic trading, and many API commands are designed for command line.
  • Front-end technologies are Boostrap, jQuery, D3, and Handlebars
  • The market server is Linux: Red Hat/Centos.
  • The back-end and market engine is a mix of C and C++; at this point, -std=c++11
  • The database is MySQL
  • The adapter communicates using FIX
  • For colocated equipment, we now recommend C++ on Linux.

How do I get involved?

The first step is to post on the forums, in the "I'm in!" section, at alphastockexchange.net. There we will discuss with you what you'd like to work on. You will then be assigned to a team, and can start contributing.

Documentation

We at Ritchie believe in great documentation. Writing docs is not the sexiest work, but it's extremely important. It must be done, and done well. Well-written docs make things easy for the consumers, and separate a good system from a great one.

If you build something, please document it here.

These docs are part of our github repo. Check it out, look for the docs/ directory, document your stuff, commit, and issue a pull request--just like anything else.

This site uses Bootstrap. If you haven't seen it, it's well worth the investment of time to learn. At this time, it holds the position as the #1 repo on Github, with more than 100,000 stars. Check out the Bootstrap Site, and there's lots of tutorials on YouTube.

For the API, we use the Handlebars templating engine. This is a little more involved, but not too bad. For the most part, this is for programmers so it's generally not a problem. Check out the Handlebars Web site for help.

Often you can copy n' paste the docs for something simlar to what you're doing, make changes as you see fit, and create a new menu item if necessary.

If you are having trouble, post in the forums under "Documentation" and we will help you.

You may be wondering, "Why aren't they using a documentation generation engine?" Although for some projects they are the perfect solution, we chose not to use one to create Ritchie docs:

  1. We feel that the inline comments required by these systems clutter the code base.
  2. They can increase the quantity of docs, but don't necessarily help to improve their quality.
  3. Inline documentation can dilute the value of source-code comments. By requiring that docs are separately written, the comments that coders add are highly focused, concise, and meaningful.
  4. The best docs are those carefully and concisely handwritten by the contributor.

We have made every effort to simplify creation of docs. We use some of the best open-source packages available, have a forum to help you if you get stuck. Writing Ritchie docs will also buff your skill set and resume with Bootstrap, jQuery, and Handlebars, and maybe help you to land that dream job.

So, please, take a moment to document your contributions, and be sure to include some relevant examples. It's really important.
